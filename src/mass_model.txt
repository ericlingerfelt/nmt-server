We are striving to provide a tool for storage, access and evaluation of mass
models such that the users can provide their own data in any one of the several
formats now used without having to rewrite the data files. Additionally, they
will be able to access this from their desktops, in effectively real-time.


What I see as our needs to implement this:

  0. We need to decide what functionality this will have and the workflow
     needed to achieve that functionality.

  1. We need to define the interface (CGI) so that work can be started on 2.
     while 3. is underway.

  2. We will need to create the new GUI to handle this separately from CINA.
     Even though some of the CINA software will be reused it may need to be
     updated and more will undoubtedly have to be developed.

  3. We will need to develop a CGI 'back-end' to handle all the data
     storage/access/manipulation work. (The current version for handling the
     Mass Models is just Java accessing a set of files and reading all the data
     from the selected file(s).  Then the Java does whatever processing needs to
     be done.)
     
     a) need to establish session management and user management software and
     interfaces.

     b) need to create data structures for storage. Will this be flat
     files, binary data, database tables, or some other method yet to be named?

     c) need to establish a method for uploading mass model data, including
     establishing (a set of) format(s) for the mass model data.

     c) need to establish storage for "bibliographic information on experimental
     mass measurements and theoretical models".

     d) need to establish management procedures and software for c) so that
     users can access/search/upload/download this information.

     e) will need to define methods for comparison and analysis of these models.
     If the software is available for an initial set of these methods it will
     need to be integrated into the CGI.

     f) (optimally) there should be some method by which a user could instantiate a
     version of their own comparison or analysis method without intervention on the
     server maintainer's part. (A plug-in if you will.)
