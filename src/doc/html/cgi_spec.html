Mass Model CGI

This information is for the mass model software suite. These are the CGI actions requested by the Java GUI. The CGI operations are performed by scripts written in PHP which access MySQL database tables.

NOTE: The returned values from the OUTPUT column, will in general be strings of the form "VALUE=xxxxxx" followed by a newline.
NOTE: Items whose background is  red    are not currently being worked on, those whose background is  pale blue  are in work and those whose background is green are completed.

CGI Property descriptions
Property Description Pattern
ID Session id ID=H48lsir8qwklnfwe90sdlkj
CREATION_DATE date and time of the creation of an object yyyy-mm-dd hh:mm:ss (2007-07-01 14:12:12)
MODIFICATION_DATE date and time of the modification of an object yyyy-mm-dd hh:mm:ss (2007-07-01 14:12:12)
DATA z,n,mass excess,uncertainty<space>repeat  
MODEL_INDEX A unique identifier for the particular model. Created when model added and stored in the database. 123
MODEL_INDICES A comma separated list of MODEL_INDEX values. 123, 456, 9987
DESCRIPTION A description of the model under consideration. This is usually supplied when the model is 'added'  
REFERENCE A reference for the model under consideration. This is usually supplied when the model is 'added'. This might a URL, a publication reference or set of references or a combination.  
AUTHOR The Author of the model (which may or may not be different from the user). string data like 'Michael Smith'
FIELD Whether the model is from experimental data (usually very small set of data) or represents a theoretical model. THEORY or EXPERIMENT
MODEL_NAME String identifying the model. Characters are constrained to uppercase letters and digits. AMDC2003
TYPE One of: USER, SHARED, PUBLIC, ALL USER
USER The account username string data like 'buckner'
PASSWORD The account password string data,SHA-1 encoded
NAME The full name of a user string data like 'Kim Buckner'
FIRST_NAME The first name of a new user string data like 'Jawaharlal'
LAST_NAME The last name of a new user string data like 'Nehru'
EMAIL A full email address string data like 'stan.lee@pow.com'
INSTITUTION We hope we get the University/lab they are working with string data like 'PNNL'
COUNTRY The (2?) character country ID/Name; see ISO for official list string data like 'US'
ADDRESS The 'mailing' address of the user string data like '1313 Mockingbird Lane, Mockingbird Heights, CA'
RESEARCH A hopefully descriptive comment on the type of research the user is doing when signing up for access. string data like 'Atomic mass stuff'
HEAR_OF_US How the user came to know of the site string data like 'Momma told me not to come'
INFO Any additional info the user wants to provide, like projects, co-workers, advising professor and so on string data like 'work with Michael Smith et.al.'
COMMENT_INDEX A unique integer identifying a comment. 99845
DATE date and time of the creation of a comment yyyy-mm-dd hh:mm:ss (2007-07-01 14:12:12)
TEXT The body of the comment.  
SUBJECT A text string that is the subject of a comment. "AMDC2003 comment about format"
OVERWRITE A flag that allows a user to 'create' a new model with the same name as an existing model, iff that user owns the model and the model is not PUBLIC or SHARED. Y or N
